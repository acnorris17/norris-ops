Decision

V5 is the final version of the Norris Utilities® Autonomous Agent Runner. All prior iterations (V3, V4) are consolidated into a single, permanent agent_runner.py with no version suffixes. V5 is production.

This decision locks the architecture. The agent runner handles task queue processing, Claude CLI orchestration, HTML output builds with full brand compliance, automated validation, Telegram notifications with typing indicators, and GitHub deployments to ops.norrisutilities.com.

Production Deployment Steps

  1. 1 Back up agent_runner.py (V3) and agent_runner_v4.py to ~/norris-agent/backups/ with dated filenames
  2. 2 Merge all V4 improvements into agent_runner.py as V5 — update version header, clean log names, remove version-specific references
  3. 3 Run python3 agent_runner.py --test-one to verify end-to-end: task build, validation, output save, and GitHub push
  4. 4 Confirm Telegram notifications fire on success and failure paths with typing indicators
  5. 5 Run --brief and --eod to verify scheduled commands work under V5
  6. 6 Run continuous loop mode for at least 3 cycles to confirm stability
  7. 7 Delete agent_runner_v4.py and any __pycache__ bytecode from ~/norris-agent/
  8. 8 Commit and push: “V5 final production release — single agent_runner.py”

V5 Finalization Checklist

  • All V4 improvements audited and confirmed present in V5 agent_runner.py
  • Known bugs resolved: typing indicator thread cleanup, HTML extraction edge cases, truncation false positives
  • Log file names standardized: agent.log and agent_error.log (no version suffixes)
  • Version string and docstring updated to V5 final production
  • All CLI flags verified: --once, --brief, --eod, --retry-failed, --test-telegram, --test-one, --feedback
  • FlexPro Armor 50% margin pricing calculations confirmed accurate

Acceptance Criteria

  • Only one agent_runner.py exists in ~/norris-agent/ — no V4 file remains
  • --test-one processes a task end-to-end: build, validate, save, push to GitHub
  • Telegram typing indicator appears during processing and stops on completion
  • Telegram notification includes file path, status, and output details
  • Continuous loop runs stable for 3+ consecutive cycles without error
  • V3 and V4 backups archived in ~/norris-agent/backups/ with date stamps
  • Clean log file names in use: agent.log, agent_error.log
