§M.2 — quickbooks@notification.intuit.com watcher (FlexPro Armor invoices)
──────────────────────────────────────────────────────────────────────────────

Trigger: New email from quickbooks@notification.intuit.com matching
subject pattern: "Invoice [N] from Norris Utilities, LLC" or
"Invoice N from Norris Utilities, LLC" (with or without brackets).

Polling cadence: 5 minutes (separate daemon from Legacy's 3-min monitor).

Pipeline:
  1. Fetch new matching emails since last_seen timestamp
  2. For each email: download PDF attachment via gmail_client.get_attachment
  3. Save PDF to ~/norris-agent/invoices/{NNNN}_{customer_short}_{YYYY-MM-DD}.pdf
  4. Parse PDF with lib/invoice_parser.py — extract:
     - invoice_number, total, line_items (P/N + description + qty + rate),
       subtotal, freight, tax, customer_name (from "Bill To" field),
       invoice_date, due_date
  5. Apply FlexPro Armor product filter (§M.5):
     - REJECT → log to invoice_skipped.jsonl with reason; route to
       human review queue (§M.5b); do NOT auto-update portal
     - ACCEPT → continue
  6. Match to portal SD via lib/invoice_sd_matcher.py:
     - Primary key: customer canonical name fuzzy match (>= 90 score)
     - Secondary: line item P/N overlap >= 50% by P/N count
     - Tertiary: date proximity within 14 days (SD ship date vs invoice date)
     - Edge case: invoice has freight line not on SD → ACCEPT match,
       flag freight as "added in QB" (variance type FREIGHT_ADDED)
     - Edge case: invoice has CC fee line not on SD → ACCEPT match,
       flag as variance type CC_FEE_ADDED
     - Edge case: invoice has additional line item beyond SD → ACCEPT
       match if P/N overlap still >= 50%, flag variance type LINE_ADDED
     - Edge case: invoice missing line item from SD → ACCEPT match if
       P/N overlap still >= 50%, flag variance type LINE_REMOVED
  7. On match:
     - Write actual invoice values to portal SD
     - Mark Invoice Sent = true
     - Set invoice_pdf_path
     - Set invoice_number (clickable link per §M.6)
     - Compute variance: rate_variance, total_variance, type tags
     - Trigger celebration (§M.7 + §S)
     - Log to invoice_ingested.jsonl
     - Tier 2 Telegram (Aaron only — CB has no Telegram per memory):
       "✅ Invoice {NNNN} ingested for {customer_short}, ${total} —
        {N} line items reconciled to SD {SD_ID}, variance: {VARIANCE_TYPE or 'NONE'}"
  8. On no match:
     - Write to portal as orphan invoice (visible in Archived Invoices)
     - Tier 2 Telegram (Aaron): "⚠ Invoice {NNNN} ingested but no SD match
        — review in Archived Invoices"
     - Add to human review queue (§M.5b) with reason NO_SD_MATCH
  9. On match with rate or total variance > 5%:
     - Set row highlight = orange in portal
     - Populate variance tracker UI with deltas
     - Tier 2 Telegram with detailed deltas
